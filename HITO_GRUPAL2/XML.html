Historia

XML (Extensible Markup Language) proviene de un lenguaje que invento IBM por los años 70, este lenguaje era GML (General Markup Language) y servía para almacenar grandes cantidades de información, como a la gente le gustó mucho lo estandarizaron y le pusieron SGML, luego en el año 89 se creó el lenguaje HTML, este lenguaje creció de una manera descontrolada por lo que no pudo cumplir todos los problemas que planteaba la sociedad global de internet, entonces ahí es donde entra XML, en el año 98 W3C empezó con el desarrollo de XML como un lenguaje de marcado flexible para la representación de datos estructurados, este lenguaje ha sido mucho más pensado y desarrollado por personas que tienen grandes conocimientos y que aún siguen trabajando en su gestación. XML pretendía solucionar problemas que surgían con el HTML.

Normas de uso

Tiene una serie de normas y estándares que se deben cumplir para garantizar que los documentos con XML sean compatibles y accesibles para la gran mayoría de aplicaciones ys sistemas y también aseguran la interoperabilidad y el intercambio de datos entre ellos, estas son algunas de sus normas:

1. Sintaxis: La sintaxis básica de XML establece la estructura de los documentos y la forma en que se representan los datos. Esta sintaxis incluye la utilización de etiquetas para representar elementos y la definición de atributos para describir características adicionales de los elementos.

2. Namespaces: Los namespaces permiten identificar de manera única los elementos y atributos de un documento XML, lo que asegura la interoperabilidad de los datos entre diferentes aplicaciones y sistemas.

3. DTD (Document Type Definition): Las DTD son una forma de definir la estructura de un documento XML y asegurar su integridad y consistencia. Las DTD definen la estructura de los documentos, los tipos de datos y las restricciones de los elementos.

4. Schema XML: El schema XML es una alternativa más avanzada a las DTD que permite una mayor flexibilidad y complejidad en la definición de la estructura de los documentos XML.

5. Reglas de accesibilidad: Las normas de accesibilidad establecen las pautas para la creación de documentos XML accesibles a personas con discapacidades.

Aplicaciones en el ámbito empresarial

Es un formato versátil y es muy utilizado en el ámbito empresarial, esto es algunas de las cosas que se incluyen en las aplicaciones:

1. Intercambio de datos: XML permite el intercambio de datos estructurados entre diferentes aplicaciones y sistemas, lo que facilita la integración y la automatización de procesos.

2. Almacenamiento de datos: XML puede utilizarse como formato de almacenamiento para datos estructurados, lo que permite su acceso y uso por diferentes aplicaciones y sistemas.

3. Representación de documentos: XML se utiliza para representar documentos electrónicos, como facturas, órdenes de compra, etc., lo que permite su intercambio y procesamiento automático.

4. Comunicación entre aplicaciones: XML es un formato de intercambio de datos común en la comunicación entre aplicaciones, especialmente en la integración de servicios web.

5. Desarrollo de aplicaciones: XML se utiliza como formato de intercambio de datos en el desarrollo de aplicaciones, lo que permite su integración y colaboración con otros sistemas y aplicaciones
